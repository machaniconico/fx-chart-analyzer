# fx-chart-analyzer 仕様書 (PRD)

FXチャート分析サイト。テクニカル分析・シグナル検出・確率ベースの価格予測・MQL EA コード生成・ブラウザ内バックテストを提供する静的SPA。

## 絶対条件
- 完全静的サイト(Cloudflare Pages でホスト)。サーバー不要。APIキー不要。課金なし。
- UIは日本語。
- 「予測」は断定しない。方向確率+予測レンジ+根拠の提示。免責事項(投資助言ではない)を常時フッター表示。

## 技術スタック
- Vite + React 18 + TypeScript (strict)
- チャート: lightweight-charts (TradingView OSS, v4系)
- スタイル: 素のCSS or CSS Modules(Tailwind不要)。ダークテーマ基調のトレーディング風UI。
- テスト: vitest(指標計算・バックテスト・MQL生成のユニットテスト必須)

## データ
- `scripts/fetch-data.mjs`: `dukascopy-node` で OHLCV を取得し `public/data/{PAIR}/{TF}.json` に保存。
  - PAIR: USDJPY, EURUSD, GBPJPY, EURJPY, AUDJPY, GBPUSD
  - TF: h1, h4, d1(h4はh1から合成してもよい)
  - 各TF直近2000本程度。フォーマット: `{ pair, tf, updatedAt, bars: [{t,o,h,l,c,v}] }`(tはunix秒)
- 生成したJSONはリポジトリにコミットする(Pagesビルドがネットワーク不要になるように)。
- `.github/workflows/update-data.yml`: 毎日 06:00 JST に fetch → 差分あれば commit & push(mainへ直push可、データ更新のみの例外)。

## 機能

### 1. チャート分析タブ
- 通貨ペア・時間足セレクタ。ローソク足チャート。
- 指標のトグル表示: SMA(20/50/200), EMA(12/26), ボリンジャーバンド(20,2σ), 一目均衡表(雲)
- サブチャート: RSI(14), MACD(12,26,9)
- 全指標は自前実装 `src/lib/indicators.ts`(外部指標ライブラリ禁止、テスト必須)

### 2. 自動分析(チャートタブ内パネル)
- `src/lib/signals.ts`: 現在のバーで検出したシグナル一覧を日本語で表示。
  - ゴールデンクロス/デッドクロス(SMA20×SMA50)、RSI 30/70 超過、MACDクロス、BBブレイク、雲の上/下、直近スイングハイ/ローからのサポレジ距離
- 総合スコア: 各シグナルを -2〜+2 で重み付け合算 → 「強い売り〜強い買い」5段階メーター表示。

### 3. 予測タブ
- `src/lib/predict.ts`: アンサンブル予測。
  - モデルA: テクニカルシグナル投票(上記スコア)
  - モデルB: 対数リターンの線形回帰(直近N本)によるドリフト推定
  - モデルC: モメンタム/平均回帰の統計(直近リターンの自己相関で切替)
  - 出力: 次の1/5/20本先の 上昇確率(%) と 予測レンジ(ATRベースの68%/95%帯)
- 表示: 確率ゲージ+チャート上にファンチャート(予測帯)描画+「なぜこの予測か」根拠リスト。
- 過去データでの的中率(方向一致率)をウォークフォワードで自己計算して正直に表示する(50%台でもそのまま出す)。

### 4. EAビルダータブ
- 戦略をフォームで組む:
  - エントリー条件(複数AND): MAクロス、RSI閾値、BBタッチ/ブレイク、MACDクロス
  - 決済: SL(pips), TP(pips), トレーリングストップ(任意), 反対シグナル決済(任意)
  - ロット、マジックナンバー
- `src/lib/backtest.ts`: 手元のヒストリカルデータでバー単位バックテスト(スプレッド固定コスト考慮)。
  - 結果: 勝率、プロフィットファクター、最大DD、取引回数、純損益(pips)、資産曲線チャート
- `src/lib/mql.ts`: 選択した戦略から MQL5 (.mq5) と MQL4 (.mq4) の完全なEAソースを文字列生成 → ダウンロードボタン。
  - 生成コードはコンパイル可能な完成品(OnTick, ポジション管理, SL/TP設定, トレーリング含む)。iCustomなし、標準指標関数のみ使用。

## 品質基準
- `npm run build` と `npm test` が緑。
- indicators/backtest/mql/predict にユニットテスト(既知値との照合含む)。
- モバイル(375px)でも崩れないレスポンシブ。
- README.md(日本語): 概要、開発手順、データ更新、免責。
- .gitignore: node_modules, dist 等。データJSONはコミット対象。

## 実装ステージ(この順で)
1. スキャフォールド + データ取得スクリプト実行(実データコミット)+ チャートタブ(指標込み)
2. シグナル分析パネル + 予測タブ
3. EAビルダー + バックテスト + MQL生成
4. 仕上げ: レスポンシブ、免責、README、GitHub Actions、テスト整備
